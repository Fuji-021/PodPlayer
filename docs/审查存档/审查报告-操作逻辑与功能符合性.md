## 🔴 P1

### 1. 后退15/前进30 按钮用了"上一首/下一首"换轨三角图标（图标语义 ≠ 实际功能）
- 操作：播放条中间金刚键左右两枚 = 后退15秒 / 前进30秒（沉浸页同款）。
- 实际：`Player.vue:146-167`（主条）+ `:694-711`（沉浸页）用 `icon-class="previous"`（step-backward `|◀`）/`"next"`（`▶|`）；注释 `A-7.7` 自认"图标暂用 previous/next、风格不一致、暂缓"。
- 偏差：三角图标表达的是"切换单集"，实际功能是"本集内 ±15/30 秒 seek"。用户怕跳集不敢点、或点了发现没换集而困惑。**项目图标库里其实已有 `seek-backward-15`/`seek-forward-30` 自绘图标（图标替换轮特意保留的），却没接上**——把这两枚 `icon-class` 换过去即可，成本极低、收益直接。
- 为何新：buglist/上轮均无；上轮只提 mediaSession 的 seek 绑定。
- **【2026-06-14 用户决定】图标保留 `previous`/`next` 不换**——确认**非命名错误**（icon-class 与文件名实一致，是"用换轨三角表达 seek"的选型）；**本条关闭、不修**。

### 2. "按最近收听"排序：切升/降序时箭头翻转，但列表顺序纹丝不动
- 操作：我的订阅页排序菜单选「按最近收听」，再点一次切方向。
- 实际：方向箭头对所有 sortBy 都渲染（`podcastLibrary.vue:27-35`）、`setSort` 翻转 `sortDir`（`:599`），但 `sortedPodcasts` 对 `lastListen` 是**提前 return 的专用分支**（`:368-379`），恒"听过的按 lastListenedAt 降序"，**完全不读 `sortDir`**。
- 偏差：箭头指示与实际排序矛盾，用户以为没生效/卡了。修：让 lastListen 分支读 sortDir，或该项干脆不显示方向箭头。
- 为何新：buglist/上轮无排序方向语义。

## 🟠 P2

### 3. 详情页/列表页"收藏"丢失 podcastId → 触发自动清理静默删除单集 ★最该先修（数据安全）
- 操作：节目详情页 / 单集详情页 / 我的下载页 / 我的收藏页 点"收藏"。
- 实际：action `togglePodcastFavorite`（`actions.js:257`）取 `track.podcastId`，但这些页面入口构造的 track **都不带 podcastId**（`podcastDetail.vue:592-604,:691-699` / `episodeDetail.vue:468-475` / `downloadsList.vue:229-240` / `favoritesList.vue:140-151`）→ 存入 `podcastId=''`。对照：主播放器心形按钮走 `currentTrack` **有** podcastId（`Player.js:1452`），存得对——同一集"从播放器收藏"和"从详情页收藏"存的不一致。
- 偏差：`prunePreviewOrphans`（`db.js:122`，启动时 `main.js:106` 调）用 `db.favorites.where('podcastId')` 保护"收藏过其单集"的预览（未订阅）节目；podcastId='' 命中 0 → 1 小时后该节目及其全部单集被删（详情页再进会回首页）。这些方法里 `ep.podcastId` 本就拿得到却丢弃。**修复成本极低：四处视图把 `podcastId: ep.podcastId` 传进 track 即可。**
- 为何新：与"软删堆积"相反（那是该删没删，这是**该留被误删**），根因是收藏写入丢字段。

### 4. 二级页「换一批」(hot/new/treasure) 点了内容完全不变
- 操作：发现页板块进二级页（`discoverList.vue`），点右上 `title="换一批"` 刷新。
- 实际：`reload(true)`→`getSectionFull`（`discover.js:169-175`）三种 type 全部确定性返回、**无 shuffle**（hot=slice、new=excludeSubbed、treasure=slice(TREASURE_START)）。
- 偏差：treasure 二级页点「换一批」得逐字相同列表；hot/new 叫「换一批」本就语义不当（应是"刷新"）。
- 为何新：buglist"reroll不换"指**首页** forYou；这是**二级页 discoverList** 上独立的 refresh 按钮、不同代码路径。

### 5. 首页「为你推荐」reroll 未排除「新上线」，三栏撞车
- 操作：首页点「再来一瓶」(forYou reroll)。
- 实际：reroll 分支只把 `subscribedMap + hot + treasure` 加 exclude（`home.vue:359-365`），**没并入 `this.newItems`**；而代码反复声明"除热门外各栏互不重复"。
- 偏差：reroll 后 forYou 可能与「新上线」重复出现。
- 为何新：与"reroll不换"不同（这是换了但与 new 撞车）。

### 6. treasure 切片口径 splitSections 与 reshuffleSection 不一致
- 实际：初次 `splitSections` 先 `slice(TREASURE_START)` 再 filter（`discover.js:147-149`）；reroll `reshuffleSection` 先 `excludeSubbed` 再 `slice(TREASURE_START)`（`:160-163`）→ 先剔已订阅使数组变短再切，丢掉腰部节目、起点漂移。
- 偏差：两路径"寻宝候选"定义不同，订阅越多偏差越大（与该文件 `TREASURE_START` 注释"统一常量避免漂移"初衷相悖）。
- 为何新：buglist/上轮无切片口径问题。

### 7. 本地"隐藏/显示播放器"快捷键(minimize)完全不生效（UI 可设可存弹成功）
- 操作：快捷键表 `minimize` 行的「快捷键」列（本地，默认 Ctrl+M），可录制/改/保存弹"已保存"。
- 实际：本地快捷键靠 Electron 菜单 accelerator 实现；`menu.js` 只为 play/next/previous/音量/like 注册（:76-117），**没有 minimize**；Window 子菜单只有静态 `role:minimize`（绑系统默认键、与配置无关）；渲染端 `handleKeydown` 也不处理本地键。全局列那格另由 `globalShortcut.js:52` 注册、能用。
- 偏差：该行「快捷键」格可设可存、显示得有模有样，按下无任何效果（只有全局列那格真生效）。
- 为何新：buglist/上轮无快捷键条目。

## 🟡 P3

8. **repeat「列表循环(on)」对播客是冗余档**：播客 `_list` 恒单元素，集末 on（空队列）→ 返回 list[0] 重播本集、one → 也重播本集，结果相同；on 与 one 对播客等价。**【2026-06-14 已定 · 删列表循环】**（播客随听随弃、放完剔除，列表循环无意义）：
   - `Player.js:1347` `switchRepeatMode` 改成 **off ↔ one（单曲循环）两档**：`this.repeatMode = this._repeatMode === 'one' ? 'off' : 'one'`（不再产生 'on'）。
   - `Player.vue:347-365` 图标态：off → `repeat`(未激活/灰)、one → `repeat-1`(激活)，去掉中间 on 态。
   - 启动读 localStorage 时把遗留 `'on'` 归一化为 `'off'`（老用户残留值）。
   - `Player.js:415-432` 的 `repeatMode==='on'` 分支可留作死代码（无害，'on' 仍是合法枚举）或顺手清理。
   - （随机键已 `v-if=false` 隐藏，符合"播客不需要随机"，无需动）

9. **未加载 howler 时按后退15/前进30 静默无反应**：`playOrPause` 有"_howler 空则懒加载"兜底（`Player.js:1203`），但 `seekBackward15/Forward30`（`Player.vue:1512`）只调 `seek()`，`seek` 在 `_howler===null` 时直接 return。重启后不先点播放直接点快进 → 没反应（两按钮兜底不一致）。

10. **OPML 导入无 feed 去重，"成功 N 档"统计虚高**：`importOpmlText`（`service.js:87-117`）无按 xmlUrl 去重，重复 feed 被订阅多次（库幂等不重复），但 `added.length` 计多次 → 弹"成功 X 档"虚高、与实际订阅数对不上。

11. **重复订阅已订阅档会清掉 newCount 角标 + 覆盖 source**：`subscribeByRssUrl`（`service.js:44`）走 `upsertPodcast`=put 整条覆盖，`podcast` 来自 parseRss 不含 newCount → 角标清零；source 被覆盖（OPML/粘贴的 manual 覆盖原 discover，发现页溯源黄点消失）；coverUrl 重置。

12. **粘贴RSS/OPML/单档导入订阅后未同步 subscribedMap**：`podcastLibrary.vue` 的 confirmAdd/runOpmlImport/runSingleRssImport 只 `loadPodcasts()` 刷新本页、**未 commit `addSubscribedPodcast`**（对照发现页/详情页订阅都 commit）；另一标签已开的发现页绿勾不即时，切页 `loadSubscribedMap` 重灌自愈。

13. **详情页"剩余时长"在 position>节目时长时显示负值乱码**：`formatProgressLabel`（`podcastDetail.vue:803-805`）`listened>30&&total>0` 直接 `formatDuration(total-listened)`、**未 `Math.max(0,…)`**；position 超 RSS 声明 duration 时为负 → "-1:-50"。对照 `historyList.vue:103` 已用 Math.max(0)。

14. **「按最新更新」排序项 lead 图标固定向下，与右侧动态方向箭头同时反向**：该项 lead 硬编码 `arrow-down-small-big`（`podcastLibrary.vue:344-348`），右侧方向箭头随 sortDir 变；asc 时一行"左↓ 右↑"相反（其它项 lead 用中性图标无此问题）。纯视觉。

15. **快捷键无冲突检测 + 保存恒提示成功**：无 conflict/isRegistered 检测；`saveShortcut`（`settings.vue:921`）无条件弹"已保存"；`globalShortcut.register` 对占用/非法 accelerator 静默返回 false。设重复/被占用键时恒弹成功但实际不工作，还可能因 `unregisterAll` 后重注册连累同批其它键。

16. **NAS「测试连接」在总开关关闭时恒报失败（即便可达）**：`testCurrentNas`→`probe()`，probe 开头 `if(!enabled) return false`；总开关 OFF（但已配好档）时点「测试连接」必弹"连不上"、与实际可达无关。对比弹窗里「测试并发现库」走临时凭据能真测——两个"测试"语义不一致。

17. **沉浸页弹窗(音量/队列/睡眠/倍速)未做主题隔离，跟随系统深浅色**：沉浸页主体硬编码深底白字（隔离 OK），但复用的弹窗用 `var(--color-body-bg)`/`--color-text`（`Player.vue:2896-2903`）；浅色模式下浅底深字漂在深色沉浸页上、不再"一个样"（仍可读，外观瑕疵）。

## 已核对、符合预期（无需动，列出以示已查）

- **播放控制**：播放/暂停在所有入口（条/沉浸页/托盘/媒体键）语义统一；倍速切歌后保持、显示与 howler.rate 一致；静音恢复到原音量（非 100%）、音量图标三档对应；睡眠定时到点真停 + 'end' 模式按真实剩余判定 + 倒计时进位；队列加入去重/插播/移除/清空/拖序正确、不双 toast；进度 seek 准确 + 续播恢复 + 已听完归零防秒跳。
- **订阅内容管理**：取消订阅三入口一致（软删 + commit removeSubscribedPodcast，残留下载/标记为设计如此）；标记时间点（短按加点/长按清空/持久/按 episodeId 隔离）；下载三态流转 + 删文件失败不删 DB；OPML 导出只导真订阅、转义正确；收藏对象是单集、列表一致。
- **导航展示**：各页跳转目标编码一致、预览态哨兵不乱跳；单集列表统一最新在前；收听历史排序/筛选/空态；统计页日期边界（YYYY-MM-DD 字典序==时间序、本地时区）；下载页计数/百分比兜底。
- **设置/新功能**：语言/外观/托盘图标即时生效 + 持久化（localStorage + electron-store）；关闭三选项分支正确；启动页逻辑；沉浸页核心交互（点 bar 进入、封面 play/pause 缩放、点名跳详情并收起、收藏/标记复用、退出解绑监听恢复滚动、播放键无圈）全部符合方案 P0 已定设计；NAS 多档管理（编辑留空沿用旧 token、一键连接重探、删当前档自动切）自洽。
