# PodPlayer 沉浸式播放页 · 设计与技术方案

> 审查/方案 session 出品（2026-06-14）。本文档**只做设计与技术拆解，不含落地代码**；落地由主 session 按本方案实施。
> 目标：点击播放 bar 进入的全屏沉浸式播放页。视觉对标 **Apple Music**（多色采样模糊背景）+ **网易云**（柔和取色）+ **第三方**（磨砂质感、横屏布局）。桌面端以**横屏（类平板）**为主形态。

---

## 一、参考图逐图拆解（先看懂别人的逻辑）

### Apple Music（图1 播放中 / 图2 暂停）
- **背景**：多色采样自封面、大面积模糊，**绝非单色**——顶部取封面暖色（橘红）、下部取冷色（青蓝），分布跟着封面走。
- **关键真相**（逆向分析）：Apple 的"流动渐变"**本质是封面自己**——把封面**过饱和**后，堆叠 4 个副本（25% / 50% / 80% / 125% 视口宽），大的两个原地旋转、小的两个沿圆轨移动，用 Pixi.js 做 twist/blur/saturation。所以它的颜色天然和谐（就是封面的色），不存在"撞色"问题——**因为根本没有人为配色，是封面被模糊放大了**。
- **封面缩放**：图1（播放）封面占满；图2（暂停）封面明显缩小、四周留白、阴影变浅。**播放↔暂停 = 放大↔缩小 + 过渡**，这是 Apple 招牌交互。
- **排版（竖屏）**：封面在上居中 → 歌名/艺人(左)+收藏★/⋯(右) → 细线进度条+时间+音质标 → 三大金刚 → 音量条 → 底部三键(歌词/AirPlay/列表)。歌名过长跑马灯滚动。

### 网易云（图3 / 图4）
- **背景单色，但不是 1:1 采样**：图3 封面是红+青强撞色，背景却只取**柔和的暗红**；图4 海蓝封面 → 柔和深蓝。
- **核心经验**：**取主色 ≠ 直接用主色**。网易云把采样色做了一层"**柔化**"——降饱和、压明度、向中性靠，所以高饱和/撞色封面也不糊脸。这就是你说的"色系不错、很柔和"的来源。

### 第三方（图5 竖屏 / 图6 横屏）
- **磨砂层**：采样色之上加一层**磨砂玻璃 + 噪点**，让色块过渡细腻、有高级感。
- **图6 横屏布局（重点参考）**：左封面（正方形圆角阴影）｜右信息区：歌名+艺人+直播标 → 细长进度条 → 三大金刚(居中) → 底部一排功能键(随机/定时/音效/列表/⋯)。**左右分栏**就是桌面宽屏该用的骨架。
- 图5 竖屏极简（只有封面+三键）说明它**强响应式**：窄则极简、宽则展开。

## 二、需求拆解（模块化）

| 模块 | 核心要求 | 难度 |
|---|---|---|
| 背景取色 | 采样封面主色系、**尽量 3 色**、**忠实封面冷暖分布**、柔化（不排除对立色） | 中 |
| 背景渲染 | 模糊纯净 + 磨砂层；先**静态**，动态留后 | 中 |
| 封面 | 居中；**播放放大/暂停缩小**带过渡；hover 无反馈；圆角 | 低-中 |
| 排版 | **封面居中放大**；控制区小而下移（占行宽 20–30%）；功能键复用原 bar | 中 |
| 进度条 | **胶囊**、更细更长、圆角匹配封面、宽度随封面 | 低 |
| 响应式 | 随窗口缩放过渡、保持比例、按钮有**最大值** | 中 |
| 全屏沉浸 | 最大化隐顶栏；ESC 退出/双击 ESC 最小化；快捷键可改；顶部 hover 提示；**不遮任务栏** | 中-高 |
| 跳转 | 点节目名/单集名 → 详情页 | 低 |
| 主题隔离 | 沉浸页固定表现，**不随深/浅色模式** | 低 |

## 三、背景方案（重点）

### 3.1 取色：node-vibrant + 和谐挑选 + 柔化
- 用 **node-vibrant**（v4.0.4，浏览器/Node/WebWorker 兼容）：一次提取 6 色板 Vibrant / DarkVibrant / LightVibrant / Muted / DarkMuted / LightMuted，每色还附可读文字色。
- **挑 3 色的策略（忠实封面 · 2026-06-14 定：取消对立色硬规则）**：
  1. 选**主色**：亮封面取 `DarkMuted`（打底更耐看），暗封面取 `Vibrant`。
  2. 第二、三色按**在封面里的权重/占比**取最显著的 swatch，**忠实还原封面色彩分布（含冷暖对比）**，不再排除"色轮对立色"。日落封面"上暖（橘红天）下冷（青蓝海）+ 中间过渡"正是封面立意，硬排除对立色就丢了这个效果。和谐靠后面的"柔化 + 大面积模糊 + 空间分隔（上暖下冷渐变）"来保证，**而非靠砍掉颜色**。
  3. 退化兜底：单色封面候选不足时，用"主色 + 其 2 个明度变体"（同色相不同亮度），**保证永远有 3 个层次**。
- **柔化（学网易云）**：选出的 3 色统一**降饱和（×0.6~0.8）+ 明度收敛到中低区间**（HSL 调整），避免高饱和糊脸。
- **CORS 关键坑**：canvas 取像素要求封面图同源或带 CORS 头，否则 canvas 被污染、取色报错。**项目已有内部 express 代理**——封面 URL 统一走代理（同源）即可绕过。落地必须确认这点。

### 3.2 背景渲染（静态，推荐"混合路线 C"）
三层由下到上叠加：
1. **底层·模糊封面**：放大的封面副本（~130% 覆盖）+ `filter: blur(60–80px) saturate(1.15)`，低透明度打底 —— 借 Apple"封面即渐变"的天然和谐质感。
2. **中层·三色径向渐变**：用 3.1 的柔和 3 色，做 2–3 个大半径 `radial-gradient` 放不同角落叠加 —— 可控的柔和色场（学网易云）。
3. **顶层·磨砂**：半透明蒙版 + `backdrop-filter: blur()` + 极轻噪点（内联 svg/png noise）—— 第三方那层高级磨砂，统一明暗、压住过曝。

> **为什么混合**：纯渐变(A)可控但略平；纯模糊封面(B)质感好但脏图不可控；**混合(C)** 兼得、且每张封面都稳。Chromium 91 对 `filter`/`backdrop-filter`/`radial-gradient` 全支持。

### 3.3 主题隔离
沉浸页**自带固定背景与文字色**（浅色文字压在中低明度背景上），用局部作用域 class 锁死，**不继承全局 `--color-*` 的深/浅切换**。无论系统深色浅色，沉浸页一个样。

## 四、封面方案
- **封面是视觉主角、做大**（2026-06-14 定）：占据画面主要高度（约容器高 **70–80%**），控制区只占其余 **20–30% 高度**，整体是"大封面 + 下方小巧控制"的桌面观感，而非移动端那种"中等封面 + 一柱控制"。
- **播放/暂停缩放**：播放 `scale(1)`、暂停 `scale(0.82)` + 阴影减弱；`transition: transform .45s cubic-bezier(.2,.7,.2,1)`（带轻微回弹）。由播放器 `isPlaying` 驱动。
- **hover 无反馈**：不加任何 hover 态（按你要求）。
- **圆角** 12–16px，**进度条圆角与之匹配**。
- 封面本身**不跳转**（防误触），跳转交给文字（见模块九）。

---

## 五、排版方案（大封面 + 控制区占高 20–30% · 已定）
> 横屏也"封面整体居中、信息在下"（非图6分栏）。2026-06-14 据反馈定稿：**封面占大头、控制区压到画面高度的 20–30%、按钮更小、三大金刚与功能键并到同一行**。

- **高度分配**：封面占容器高 **70–80%**（视觉主角）；下方控制区**只占 20–30% 高度**，紧凑。
- **纵向层次（从上到下）**：
  1. **大封面**（居中、固定高度槽内 scale）
  2. **节目名 + 单集名**（左，排版保留当前样式）｜**收藏 ♥**（右，单个，**去掉 ⋯**）
  3. **进度条**（细、与封面同宽、**首尾不放时间**）
  4. **一行控制**：三大金刚（后退15s / 播放暂停 / 前进30s）**与 4 个功能键并在同一行**
- **按钮更小**：金刚键与功能键都比上一版再小一圈、整体面积收敛；**播放键无圆圈包围**；图标与点击动效（`tap`）**复用原 bar**。
- **合并一行的排布**：建议三大金刚居中、4 个功能键分列其侧（或金刚居中、功能键靠右），整行水平居中、紧凑；具体间距主 session 调，不显空也不拥挤。
- **功能键 = 只保留 4 个（2026-06-14 定）**：**播放列表（队列）/ 睡眠 / 倍速 / 音量**。点击行为**完全复用原 `Player.vue` 逻辑**——弹出各自的调节小弹窗（队列面板 / 睡眠菜单 / 倍速菜单 / 音量调节）。**删除** 循环 / 随机 / 倒序 / 标记 / 歌词 等，不进沉浸页。
- **封面缩放不挤压布局**：封面在**固定高度槽内** scale，下方控制位置不随之跳动（否则暂停时整列上下抖）。
- **窄窗/竖屏**：同一套纵向布局天然适配。

## 六、进度条方案
- **更细更长的胶囊**：高 **3–4px**（比上一版更细，避免"粗短"）、`border-radius: 999px`、长度与封面同宽——长条利于两三小时的长节目精确拖动。圆角呼应封面。
- **进度条首尾不放时间**（2026-06-14 定）：两端**完全不显示**已播/总时长；**只在 hover/拖动时**，在指针对应位置浮出该点的时间气泡。保持进度条干净。
- 已播/未播两段 + 可拖 thumb（hover 放大）。与主页面细线明显区分。

## 七、响应式方案（随窗口缩放）
- **整体有过渡、不呆板**：关键尺寸用 `clamp(min, 优选vw/vmin, max)`，让缩放平滑。窗口 resize 时若逐属性 `transition` 会抖，建议整体用 `transform: scale` 缩放或仅对字号/间距用 clamp。
- **比例保持**：封面 `aspect-ratio: 1`；字号、按钮全用 clamp。
- **按钮最大值（你的硬要求）**：clamp 的 `max` 即上限，防止拉满屏时按钮巨大变丑。例：金刚键 `clamp(44px, 5vmin, 72px)`、封面 `min(70vh, 56vw, 640px)`。

## 八、全屏沉浸 + 退出（技术核心）

**关键决断：不用 OS 独占全屏（`setFullScreen` / HTML5 Fullscreen），改用「无边框窗口 + 最大化到工作区」。** 理由：
- `setFullScreen(true)` 会**盖住任务栏**；且 HTML5 Fullscreen 的 **ESC 退出是浏览器内置、不可定制**——与你"ESC 可自定义 / 双击 ESC 最小化"直接冲突。
- `win.maximize()` 在 Windows **默认最大化到工作区（不盖任务栏）**，正好满足你"不干扰任务栏"。

做法：
1. **进入沉浸**：`win.maximize()` + 渲染端**隐藏自绘顶栏**（单集名/logo/最小化-最大化-关闭三键，本就是 frameless 自绘的，`display:none` 即可）→ 顶栏消失、内容铺满工作区。
2. **ESC 完全自定义**（因为没用 HTML5 fullscreen，行为我们说了算）：渲染端监听 keydown——单击 Esc 退出沉浸（恢复普通窗口+顶栏）；**短时间内双击 Esc** → `win.minimize()`。
3. **快捷键可改**：做一个"沉浸退出键"设置项（默认 Esc），监听时读配置 → 留出自定义入口。
4. **退出提示 = 跟随鼠标的小弹窗**（2026-06-14 改）：鼠标移到顶部边界（≤8px）触发，弹一个**跟随鼠标 x 位置**的小气泡"按 Esc 退出沉浸"，**贴近顶部、不挡封面**，且 **clamp 在窗口内不越界**（靠近左右边时气泡不溢出）。离开边界即隐藏。**不要**做成固定居中的大浮层（会挡封面，上一版预览就是反例）。
5. **不扰任务栏**：用 maximize 而非 fullscreen；**切勿**设 `alwaysOnTop` / `kiosk` / `skipTaskbar`（那才会强势盖住或抢焦，导致你担心的"无法退出/遮任务栏"）。

> 你那句"整屏只有我的软件、顶栏消失，但任务栏该在就在、ESC 可退" —— 表面像"独占全屏"，实则是「最大化到工作区 + 隐藏自绘顶栏」。这条路把张力全解了。

## 十、技术选型与兼容性（Chromium 91 / Vue 2）
- **取色**：node-vibrant。注意 v4.x 可能用较新语法，需确认 webpack4 + babel 能转译；不行则退 3.x（UMD）或 `@vibrant/core`。取色**仅在切歌时算一次**并缓存到该单集。
- **背景**：CSS `filter` / `backdrop-filter` / `radial-gradient`，Chromium 91 全支持。
- **性能**：大面积 blur 吃 GPU，但**静态背景只在切歌重绘**，开销可控；**绝不要**对动态 60fps 做实时 blur（故一期静态，动态留 P2）。
- **CORS**：封面走项目内部 express 代理同源，解决 canvas 污染。
- **接入**：沉浸页做成**全屏 overlay 组件**（或独立路由），播放 bar 点击 toggle；落地时确认与现有 Player 组件 / 路由 / howler 播放状态的对接（主 session 域）。

### 十·补 不干扰原有逻辑（硬约束，务必遵守）
沉浸页是**叠加的 UI 皮肤**，不得改动既有功能行为：
- **复用、不重写**：播放/暂停、后退15/前进30、播放列表/睡眠/倍速/音量等，**全部调用 `Player.vue` 现有的同一套方法与状态**（同一个 howler 实例、同一份 Vuex/播放器状态）。**严禁**为沉浸页另起一套播放逻辑或复制状态。
- **不破坏原播放 bar**：原底部播放 bar 及其所有按钮、快捷键、逻辑保持不变；沉浸页与它**共享状态、双向同步**（沉浸页里暂停，退出后原 bar 也是暂停态）。
- **可开可关、可回退**：用开关/路由控制，关掉后一切如常；最好用 feature flag 包裹，出问题一键回退。
- **样式隔离**：沉浸页样式独立作用域（scoped / 命名空间 class），**不污染全局**、不影响其他页面与主题。
- **窗口状态可恢复**：进入沉浸 `maximize` 前**记录原窗口状态**，退出**精确还原**（大小/位置/是否最大化）；ESC 等监听在退出时**务必解绑**，不泄漏到普通界面。

## 十一、决策（已定 2026-06-14）
1. 背景视觉：**C 混合**（模糊封面打底 + 三色柔和渐变 + 磨砂层）。✅

## 十二、落地分期建议
> **开工前先切分支**：主 session 从当前开发分支切出 `feature/immersive-player`，所有沉浸页改动在该分支进行（独立开发、不污染主线、做坏可弃）。

- **P0（视觉骨架）**：静态混合背景(取色+柔化+磨砂) + 大封面居中布局(控制区占高20–30%、金刚与功能并一行) + 封面播放/暂停缩放 + 细胶囊进度条(首尾无时间) + 复用原 Player 方法 + 响应式 clamp。**先做这步给你看效果。**
- **P1（沉浸交互）**：全屏(maximize+隐顶栏) + ESC 自定义退出(单击退/双击最小化) + 顶部 hover 提示 + 快捷键配置 + 文字跳转 + 主题隔离。
- **P2（锦上添花）**：动态流体背景（Apple 式多副本旋转 / Pixi.js，或 CSS 缓动平移渐变）。
- **P3（远期·高难，短期不做）**：图标"从哪来到哪去"——展开沉浸页时控制图标从原播放 bar 位置**动画飞入**当前布局、收起时**飞回原位**（FLIP 技术：记录首末位置 + transform 过渡）。技术可行但工程量大、需两套布局元素一一对应；视沉浸页完善进度再定。
